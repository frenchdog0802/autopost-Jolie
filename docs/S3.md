# S3 Bucket Configuration

## Public read

Uploaded images must be reachable by OpenAI Vision and Meta Graph APIs.

- Block public access settings should allow bucket policy or object ACL used by the app (`public-read` on upload).
- Example public URL format: `https://<bucket>.s3.<region>.amazonaws.com/uploads/<uuid>.jpg`

## Lifecycle rule (deployment-04)

Create a lifecycle rule on prefix `uploads/` to expire objects after **1 hour** as backup cleanup if application delete fails.

AWS CLI example:

```bash
aws s3api put-bucket-lifecycle-configuration \
  --bucket YOUR_BUCKET \
  --lifecycle-configuration file://lifecycle.json
```

`lifecycle.json`:

```json
{
  "Rules": [
    {
      "ID": "ExpireUploadsAfter1Hour",
      "Status": "Enabled",
      "Filter": { "Prefix": "uploads/" },
      "Expiration": { "Days": 1 }
    }
  ]
}
```

Note: S3 lifecycle minimum is 1 day for standard expiration; for 1-hour expiry use S3 Lifecycle with transition rules or rely on application delete + this documented backup policy per your AWS account capabilities.

Verify the rule exists in AWS Console after creation.
